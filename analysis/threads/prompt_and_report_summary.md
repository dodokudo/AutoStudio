## 1. Threads運用プロンプト全文

### THREADS_OPERATION_PROMPT（threadsOperationPrompt.ts）

投稿生成時に使用されるマスタープロンプト。以下が全体構成：

#### 文体DNA（工藤さんの文体ルール）

**リズム・テンポ設計:**
- 実績を冒頭2行目に配置（インパクト最優先・最重要）
  - 例：「2ヶ月でフォロワー2500名、167万インプ達成した832件のデータ分析したら...」
- 短文でフック → 2行目に実績 → 理由解説の流れ
- 関西弁要素の絶妙配置：「マジで」（驚き時）「やばい」（効果強調）「だるくない？」（共感誘発）
- キャッチーな表現：「爆伸び」「時代遅れすぎます」
- 改行は柔軟に調整（読みやすさ重視）
- コメント間の繋ぎ文言を必ず入れる + ▼記号
- 音声入力風の自然な流れ：「〜なんですよね」「〜じゃないですか」多用
- 論理の接続詞を使う：「つまり、」「でも、」
- **文末表現：「です・ます調」を厳守**
  - 完全NG表現：「〜てる」「〜ない」で終わる文、ため口

**体験談挿入パターン:**
- 失敗からの逆転：「僕も最初は○○だったけど」
- 失敗の具体数値は削除（簡潔に「停滞してました」）
- 冗長な説明は削除
- 「〜なんですよね」の不要な文末表現を削除して簡潔に

**共感・ツッコミ要素:**
- 読者の心の声代弁：「○○って感じたことないですか？」
- 煽り表現は控えめに（「別次元」「人生変わる」は削除）
- 「一緒に○○」系のCTAは使わない（上から目線回避）

#### 投稿パターン（7種類）

| パターン | 名称 | 心理的流れ |
|---------|------|----------|
| 1 | 数値・効率化型 | 現状不満→可能性提示→方法開示→行動促進 |
| 2 | 危機感煽り型 | 現状認識→危機感醸成→解決策提示→安心感付与 |
| 3 | 逆説・常識破壊型 | 常識提示→強烈否定→真実暴露→新常識定着 |
| 4 | 比較・使い分け型 | 混乱状態→整理→判断基準明示→選択支援 |
| 5 | 裏技・秘密ノウハウ型 | 好奇心喚起→驚きの事実→理解促進→習得支援 |
| 6 | 失敗談・注意喚起型 | 失敗開示→共感獲得→学習内容→予防策提示 |
| **7** | **否定・権威型（門口パターン）【最強】** | 衝撃フック→共通の敵否定→読者の間違い指摘→権威引用→解決策→CTA |

#### 否定対象リスト（工藤さん用）
- いいね周り・フォロー周り・コメント周り（時間の無駄）
- 相互フォロー・インサイト祭り（アカウント壊れる）
- 短文投稿の量産（価値が薄い）
- いいね数・コメント数ばかり追いかける（本質じゃない）
- いいね＝閲覧数と勘違い（全く別の指標）
- フォロワー数伸びれば売上上がると思ってる（伸びない）
- 閲覧数伸びればフォロワー伸びると思ってる（一致しない）
- フォロワー数伸びてもLINE登録数は伸びない（別の話）
- 冒頭1行目を適当に書いてる（ここが全て）
- 日常の投稿しかしていない（一貫性ない）
- バズった投稿の表面だけ真似してる（本質じゃない）
- フォロー・いいね・コメント周りを推奨する発信者から学んでる（間違った師匠）
- 他人の投稿に自分からアクションして伸ばそうとしてる（逆効果）

```json
{
  "posts": [
    {
      "planId": "plan-01",

      "scheduledTime": "07:00",

      "comments": [
        "コメント欄1: 必ず400文字以上、最大500文字（体験談+基本ノウハウ）",
        "コメント欄2: 必ず400文字以上、最大500文字（応用+注意点+行動促進）"
      ]
    }
  ]
}
```

**文字数配分（厳守）:**
- メイン投稿：50-150文字（インパクト重視、短く刺す）
- コメント欄1：必ず400文字以上、最大500文字（体験談+基本ノウハウ）
- コメント欄2：必ず400文字以上、最大500文字（応用+注意点+行動促進）
- 合計：900字前後が目標

**改行ルール:**
- JSON文字列内で改行は「\n\n」で表現
- 基本は1行1文。改行は最小限に
- リスト記号は「・」（中黒）のみ使用

## 4. 工藤さんの文体DNA（まとめ）

### 絶対ルール
1. **絵文字厳禁**（記号▼■、数字は可）
2. **です・ます調を厳守**（全文通して統一）
3. **実績は冒頭2行目に配置**（最重要）
4. **コメント欄は400文字以上**（300文字以下は絶対NG）
5. **リスト記号は「・」のみ**（■◆は使わない）
6. **数値表記：カンマなし、「名」**（1600名）
7. **「1日」単位で統一**（週単位は使わない）
8. **箇条書きは①②③**（「1つ目が、」は使わない）
9. **繋ぎ文言 + ▼記号を必ず入れる**

### 使うべき表現
- 「マジで」「やばい」「だるくない？」（関西弁要素3箇所以上）
- 「爆伸び」「時代遅れすぎます」（キャッチーな表現）
- 「つまり、」「でも、」（論理の接続詞）
- 「〜なんですよね」「〜じゃないですか」（音声入力風）
- 「僕も最初は〜」（体験談）

### 使ってはいけない表現
- ため口（「〜てる」「〜しろ」「〜してみて」で終わる文）
- 「別次元」「人生変わる」（煽り表現）
- 「一緒に○○しましょう」（上から目線CTA）
- 保存数・滞在時間の具体的数値（測定不可のため）
- フォロー促進のCTA（@kudooo_ai への誘導なし）

## 5. 絶対禁止事項

1. **絵文字の使用厳禁**
2. **学習事例と同じテーマの使用禁止**：改行位置、コメント欄活用、2段階ピーク、投稿時間帯、バズる型、プロフィール最適化、CTA設計、完璧より頻度、バズとフォロワーの真実

5. **「一緒に○○しましょう」系のCTAは使わない**

7. **「今すぐ試さないと〜」系の強い煽りは削除**
8. **コメント300文字以下は絶対NG**

## 8. プロンプトビルダーの仕組み（promptBuilder.ts）

`buildThreadsPromptPayload()` が以下のデータをBigQueryから並行取得してプロンプトに組み込む：

1. **アカウントサマリー**: フォロワー数、プロフ閲覧数、増減
2. **自社トップ投稿**: 直近期間のインプレッション上位10件（コメントデータ付き）
3. **競合ハイライト**: 競合投稿のインプレッション上位5件
4. **トレンドトピック**: ジャンル別のフォロワー増減トレンド
5. **テンプレートサマリー**: 48時間以内の投稿とのマッチングによるテンプレート別パフォーマンス
6. **競合選抜投稿**: AI系20本 + 非AI系30本 = 50本（ティアS/A/B、ランダムサンプリング）
7. **自社勝ち投稿**: pattern_win/niche_hit/hidden_gemの上位10本
8. **門口さん投稿**: ティアS/Aの長文500字以上から5本
9. **Web検索結果**（Tavily）: 最新AIニュース + 実践HowTo

### 投稿生成フロー（claude.ts）
1. `buildBatchPrompt()` でコンテキスト + KUDO_MASTER_PROMPT を結合
2. Claude API（claude-sonnet-4-5-20250929）に1回のリクエストで10本一括生成
3. JSON形式で返却 → バリデーション → サニタイズ
4. temperature: 0.9（多様性重視）
